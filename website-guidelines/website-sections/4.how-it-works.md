# How It Works Specifications

## Content
A 3-step guide explaining the user journey:
1. **Solve Puzzles** - Complete logic-based challenges
2. **Earn Cookies** - Receive 1-5 cookies per solved puzzle
3. **Unlock Python** - Transition to real Python coding

Each step should include:
- Step number
- Title
- Brief description
- Visual representation (icon, illustration, or diagram)

## Animations / Effects
### Step Reveal Animation
- Type: [Framer Motion]
- Reference: Steps animate in sequentially from left to right
- Code:
```tsx
{steps.map((step, index) => (
  <motion.div
    key={step.id}
    initial={{ opacity: 0, x: -20 }}
    whileInView={{ opacity: 1, x: 0 }}
    viewport={{ once: true }}
    transition={{ delay: index * 0.2 }}
  >
    <StepCard step={step} />
  </motion.div>
))}
```

### Connector Animation
- Type: [Framer Motion]
- Reference: Connecting lines/arrows draw in after steps appear
- Code:
```tsx
<motion.div
  initial={{ scaleX: 0 }}
  whileInView={{ scaleX: 1 }}
  viewport={{ once: true }}
  transition={{ duration: 0.5, delay: 0.3 }}
  className="origin-left"
>
  <Connector />
</motion.div>
```

### Step Number Pulse
- Type: [CSS / Framer Motion]
- Reference: Step numbers have subtle pulse to draw attention
- Code:
```tsx
<motion.div
  animate={{ scale: [1, 1.05, 1] }}
  transition={{ duration: 2, repeat: Infinity }}
  className="step-number"
>
  {stepNumber}
</motion.div>
```

### Hover Effect on Steps
- Type: [Framer Motion]
- Reference: Cards lift and glow on hover
- Code:
```tsx
<motion.div
  whileHover={{ y: -8, scale: 1.02 }}
  transition={{ duration: 0.2 }}
  className="cursor-pointer"
>
  <StepCard />
</motion.div>
```

## Design Instructions
- **Spacing:** Generous spacing between steps (48-64px on desktop)
- **Flow Direction:** Left to right on desktop, top to bottom on mobile
- **Step Numbers:** Large, prominent circular badges with gradient backgrounds
- **Icons:** Large, colorful icons (80px) that clearly represent each step
- **Connectors:** Visual flow indicators (arrows or lines) between steps
- **Responsive:** Stack vertically on mobile with connectors hidden or vertical
- **Visual Hierarchy:** Step numbers should be most prominent, followed by icons, then text
- **Color Coding:** Each step can have its own color theme for easy recognition
