# Benefits Specifications

## Content
A visual grid highlighting four key benefits:
1. **Logic First:** Build a strong foundation before touching code
2. **Gamified Rewards:** Earn cookies for every solved puzzle
3. **Python Ready:** Smooth transition to real-world programming
4. **Track Progress:** Always pick up where you left off

Each benefit should include:
- Icon or illustration
- Title
- Brief description
- Visual element (illustration, icon, or graphic)

## Animations / Effects
### Grid Stagger Animation
- Type: [Framer Motion]
- Reference: Benefit cards fade in with stagger effect
- Code:
```tsx
<motion.div
  initial="hidden"
  whileInView="visible"
  viewport={{ once: true }}
  variants={containerVariants}
  className="grid"
>
  {benefits.map((benefit, index) => (
    <motion.div
      key={benefit.id}
      variants={itemVariants}
      custom={index}
    >
      <BenefitCard benefit={benefit} />
    </motion.div>
  ))}
</motion.div>

const containerVariants = {
  hidden: { opacity: 0 },
  visible: {
    opacity: 1,
    transition: {
      staggerChildren: 0.1
    }
  }
}

const itemVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: (index) => ({
    opacity: 1,
    y: 0,
    transition: { delay: index * 0.1 }
  })
}
```

## Design Instructions
- **Grid Layout:** 2x2 on tablet, 4 columns on desktop, single column on mobile
- **Card Consistency:** Equal height cards for visual balance
- **Icon Size:** Large, prominent icons (64px minimum)
- **Color Coding:** Each benefit can have its own accent color
- **Spacing:** Comfortable gaps between cards (24px)
- **Visual Elements:** Include engaging graphics that reinforce the benefit message
- **Typography:** Clear hierarchy with bold titles and readable descriptions
- **Responsive:** Cards stack vertically on mobile with full width
- **Accessibility:** High contrast, descriptive icons, clear text
