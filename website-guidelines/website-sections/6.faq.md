# FAQ Specifications

## Content
Common questions about CodeCookies:
1. **Age Appropriateness:** What age is this suitable for?
2. **Progress Saving:** How is progress saved and can it be lost?
3. **Safety:** What safety measures are in place? (COPPA/GDPR compliance)
4. **Additional Questions:** 
   - How do cookies work?
   - Can parents track their child's progress?
   - What devices are supported?

Each FAQ item should include:
- Question
- Answer (expandable/collapsible)

## Components to Use
### FAQ Container
- Source: [custom]
- Description: Accordion-style FAQ section with expandable questions
- Code:
```tsx
<section className="py-20 bg-gray-50 dark:bg-gray-800">
  <div className="container mx-auto px-4 max-w-3xl">
    <h2 className="text-4xl font-bold text-center mb-12">Frequently Asked Questions</h2>
    <div className="space-y-4">
      {faqs.map((faq) => (
        <FAQItem key={faq.id} question={faq.question} answer={faq.answer} />
      ))}
    </div>
  </div>

```

### FAQ Item Component (Accordion)
- Source: [shadcn Accordion]
- Description: Expandable/collapsible FAQ item with question and answer
- Code:
```tsx
<Accordion type="single" collapsible className="w-full">
  <AccordionItem value={id}>
    <AccordionTrigger className="text-left text-lg font-semibold">
      {question}
    </AccordionTrigger>
    <AccordionContent className="text-gray-600 dark:text-gray-400 pt-2">
      {answer}
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

### FAQ Card Alternative
- Source: [custom]
- Description: Card-based FAQ with hover states
- Code:
```tsx
<Card className="p-6 hover:shadow-lg transition-shadow">
  <button
    onClick={() => setIsOpen(!isOpen)}
    className="w-full flex items-center justify-between text-left"
  >
    <h3 className="text-lg font-semibold">{question}</h3>
    <ChevronDown className={`w-5 h-5 transition-transform ${isOpen ? 'rotate-180' : ''}`} />
  </button>
  {isOpen && (
    <motion.div
      initial={{ height: 0, opacity: 0 }}
      animate={{ height: 'auto', opacity: 1 }}
      exit={{ height: 0, opacity: 0 }}
      className="mt-4"
    >
      <p className="text-gray-600 dark:text-gray-400">{answer}</p>
    </motion.div>
  )}
</Card>
```

## Animations / Effects
### Accordion Expand/Collapse
- Type: [Framer Motion / shadcn]
- Reference: Smooth expand/collapse animation with height transition
- Code:
```tsx
<motion.div
  initial={false}
  animate={{ height: isOpen ? 'auto' : 0 }}
  transition={{ duration: 0.3 }}
  className="overflow-hidden"
>
  <div className="pt-4">
    {answer}
  </div>
</motion.div>
```

### Chevron Rotation
- Type: [CSS / Framer Motion]
- Reference: Chevron icon rotates when FAQ item is expanded
- Code:
```tsx
<motion.div
  animate={{ rotate: isOpen ? 180 : 0 }}
  transition={{ duration: 0.3 }}
>
  <ChevronDown className="w-5 h-5" />
</motion.div>
```

### Stagger Reveal
- Type: [Framer Motion]
- Reference: FAQ items fade in sequentially on page load
- Code:
```tsx
{faqs.map((faq, index) => (
  <motion.div
    key={faq.id}
    initial={{ opacity: 0, y: 20 }}
    whileInView={{ opacity: 1, y: 0 }}
    viewport={{ once: true }}
    transition={{ delay: index * 0.1 }}
  >
    <FAQItem faq={faq} />
  </motion.div>
))}
```

### Hover Effect
- Type: [CSS]
- Reference: FAQ items have subtle background color change on hover
- Code:
```tsx
<AccordionTrigger className="hover:bg-gray-100 dark:hover:bg-gray-700 rounded-md px-4 py-3 transition-colors">
  {question}
</AccordionTrigger>
```

## Design Instructions
- **Layout:** Single column, centered, max-width container (768px)
- **Spacing:** Comfortable spacing between FAQ items (16-24px)
- **Typography:** Large, readable question text (18px+), clear answer text
- **Interaction:** Clear visual feedback when hovering or clicking
- **Icons:** Chevron or plus/minus icons to indicate expand/collapse state
- **Accessibility:** Keyboard navigation support, ARIA labels, focus states
- **Responsive:** Full width on mobile, centered with max-width on desktop
- **Visual Hierarchy:** Questions should stand out more than answers
- **Child-Friendly:** Large clickable areas, clear visual states
